_ 湯谷温泉にいってきた(1)

GW を利用して温泉へ行くことに．場所は静岡と岐阜の境にある湯谷温泉へ．8 時過ぎに出発して，一路，浜松へ．東名高速は GW といいつつも，一応平日ということもあって，岡崎あたりで若干渋滞はしたものの，それほど車も多くない（っていうか，普段どれくらいか知らないけど）．三ケ日 IC で一度降りて，下道で浜松へ向かう（ ETC の通勤割引を利用したので，ちょうどここまでが 100 km 手前）．

三ケ日といえばみかん．道路沿いには，いろんなところでみかんを売っている．時期的には甘夏もおいしい時期なんだろうか．あとで買いにいこうかとか思いながら，10 時半頃に浜松市内へ．

浜松に入ると，うなぎの看板がたくさん目に入る．せっかく，浜松にきたんだし，昼ごはんはやっぱりうなぎにする？ってことで，本屋に入って，いい店がないかを探してみる．ガイドブックに掲載されている，「かんたろう」というお店を選択．店についたのが 12 時前．ちょうど昼時ということもあって，駐車場は満車．どうするかなぁ，と思ってたらちょうど 1 台出ていくところで，運よく店に入ることができた．

うなぎは関西風の焼き方で身はふっくらしつつも，表面はかりっとしていて，あっという間に食べ終わってしまった．もともとはうなぎの予定はなかったけど，思いがけずうまいうなぎにめぐりあえてよかった．機会があればまた是非食べに行こう．

おなかもいっぱいになったところで，今日の目的地である湯谷温泉を目指すことに．途中，野菜の直売所で，いちごやら甘夏を購入．なんだかんだで寄り道しながら向かったけど，14 時過ぎには目的地，湯谷温泉の泉山閣に到着．チェックインをすますと，ロビーで抹茶がいただける．部屋につくとトンビがちょうど何かもらっていたようで，たくさん飛んでいた．

部屋で一休みしたところで，時間はまだ 15 時にもなっていない．とりあえず，今日も暑かったのでまずは温泉で汗を流すことに．露天風呂でぼんやりと外を眺めながら，ゆったりとした時間を過ごす．部屋に戻って冷たい水を飲んだら，少し散歩をしてみることにした．

夕飯は，結構たっぷりな量が出てきておなかいっぱいに．どれもとてもおいしかったです．

就寝前に再び露天風呂へ．10 時過ぎということで，他の入浴者もいなくて，無料でいただくことのできる日本酒を飲みながら，のんびり時間を過ごしました．

_ 湯谷温泉にいってきた(2)

朝食をいただき，8 時 40 分からトンビの餌付けタイムを観覧．何十羽ものトンビがえさをもらいにとんでくる．あれだけの数を見ることはそうないだろうし，なかなか面白かった．朝，もう一度温泉に入って，のんびりとチェックアウト．

鳳来寺パークウェイを通って，鳳来寺へ．もみじの木がたくさんあったので，紅葉の時期はとてもきれいなんだろうなぁ．新緑のもみじもこれはこれでよかったけど．ちなみに，葉が赤いものもあったんだけど，あれは種類がちがうのかなぁ・・・

山を下って，恵那方面へと向かう．昼時に道の駅，アグリステーション「なぐら」でちょっと休憩・・・と思ったら，結構混雑してる．五平餅を食べようかと思ったけど，結構な待ち時間．そして予想以上にでかい五平餅だったので，店内で食べることに．落花生とエゴマの五平餅をそれぞれ注文．2 人でおなかいっぱいになる量でした．帰りの目的はその先にある道の駅，らっせいみさとでそばを食べることだったので，あまりしっかりおなかいっぱいになっても困るわけで．その後，目的の道の駅に到着し，そばを注文．こちらもなかなかおいしいそばでした．

その後，国道 19 号を走って帰宅．途中，小牧市内で渋滞に巻き込まれたものの，それでも明るいうちに帰宅できた．せっかくの旅も渋滞でぐったりとかだと，もったいないしね．

今回の温泉旅行も満足な旅だったなぁ．やっぱり温泉っていいですね．

_ OSX Leopard から Screen Sharing を使って Ubuntu に接続する

OSX Leopard には標準で Screen Sharing という VNC クライアントが利用できる．Ubuntu 側のリモート・デスクトップの設定で，セキュリティの「パスワードの入力を要求する」のチェックを外していると，Screen Sharing からは接続できない．

Screen Sharing は，/System/Library/CoreServices/Screen Sharing から呼び出すこともできるし，Finder で cmd + k，vnc://xxx.xxx.xxx.xxx で接続することも可能．

どうでもいいけど，OSX から Ubuntu へリモートデスクトップで接続し，その上で動作している Windows に接続とか，何やってんだか…

_ お届けものが届きません

Amazon で注文した商品が昨日発送されて，配達状況を Web で確認．再配達申し込みして，夕方さくさくと帰宅．20 時から 21 時の間にくるはずだった…

が，どうやら自分が再配達の申し込みをちゃんとやってなかったみたいで（確認メールがきてなかった），お届けものは届きませんでした．まぁ，急ぎでもなかったので，改めて Web で再配達を申し込み．今度はちゃんと確認メールもきた．

なんか，くだらないミスがここのところ続いてるなぁ．

_ Windows 環境で Perl を exe にしようとしたらエラーになった

2 年前くらいに誰かが作った Perl を修正して，PAR で exe にしようとしたらなんかエラーが出た．

プロシージャ エントリ ポイント Perl_sv_2iv_flags がダイナミック リンク ライブラリ perl58.dll から見つかりませんでした。

よく分からないのでひとまず検索．Re: Problems with ActiveState PAR/PP -- procedure entry point error に書いてあることをやったら動いた，というのがいくつか見られたので試してみたところ，無事に動作した．

普段は coLinux や リモートの Linux マシンなんかで Perl を使うことが多いので，この環境で Windows 用の exe も作れたら楽なんだけどなぁ・・・
