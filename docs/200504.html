2005-04-02

_ tDiary 2.0.1

tDiary 2.0.1がリリースされたので，このサーバでもアップデートを実施しました．しばらくの間ツッコミができない状態だったらしく，おがみ屋さんが教えてくれてやっと気づくという，無能さを発揮してしまった．

テスト環境ではツッコミ機能は復活しているので，本番のこちらでもおそらく大丈夫なはず…．ひとまず書き込みが可能かテスト．

さて，アップデート時に簡単なミスをしてしまったのでついでに書いておこう．以前はデフォルトのtDiaryスタイルで記述していたのですが，今年からWikiスタイルに記述方法を変更しました．スタイルを変更した場合は，Wikiスタイルの場合，tdiary/以下にwiki_parser.rbとwiki_style.rbをコピーする必要があるのですが，それをやらずにページを表示しようとすると，

(eval):1:in `restore': compile error
(eval):1: parse error
::new( headers['Date'], headers['Title'], body, Time::at( headers['Last-Modified'].to_i ) )
   ^ (SyntaxError)

と，エラーとなってしまいます．スタイルをデフォルトから変更している場合は要注意です…．

_ Firefox 1.0.3 Release!!

Firefox 1.0.3がリリースされました．以下のセキュリティFixが中心のようです．既知の脆弱性のページでは以下の修正があげられています．

  • [moderate] MFSA 2005-33 Javascript "lambda" replace exposes memory contents
  • [high] MFSA 2005-34 javascript: PLUGINSPAGE code execution
  • [moderate] MFSA 2005-35 Showing blocked javascript: popup uses wrong privilege context
  • [high] MFSA 2005-36 Cross-site scripting through global scope pollution
  • [critical] MFSA 2005-37 Code execution through javascript: favicons
  • [moderate] MFSA 2005-38 Search plugin cross-site scripting
  • [critical] MFSA 2005-39 Arbitrary code execution from Firefox sidebar panel II
  • [moderate] MFSA 2005-40 Missing Install object instance checks
  • [critical] MFSA 2005-41 Privilege escalation via DOM property overrides

修正項目のうち3つは「critical(緊急)」とのことですから早急にアップデートされることをお勧めします．日本語版は今のところまだ公式サイトからリンクはされていないようですが，ダウンロードは可能なようです．

_ Ajaxを利用した地図サービス

Ajaxを利用したサービスとして代表的なものはGmailやGoogle Maps，そしてGoogle Suggestなどがありますが，日本でもgooがgoo地図でAjaxを利用した地図サービスを提供しはじめました．NDO::Weblogによると，Ajaxとは，

JavaScript の XMLHttpRequest を使って XML 文書その他を取得して、それを Dynamic HTML で描写するという、既存の技術のあわせ技。Google Maps や GMail、Google Suggest なんかで日の目を見て最近はアルファな geek の注目のテクニック...

と，いうことです．

ためしに利用してみたのですが，これはなかなか使いやすいのではないでしょうか．これまで主に利用してきた地図サービスといえばMapionなのですが，これまでは表示したい部分をクリックしてその部分が中心にくるといったものでしたが，goo地図は，マウスをドラッグして地図を動かすので，より使いやすいと思います．

Google Mapsと地図を動かす方法が逆になるので，一番最初に動かしたときはあれっと思いましたが，すぐに慣れてしまいました．動きとしてはGoogle Mapsの方が好みなのですが，国内の地図サービスではgooが最初とのことで，今後，goo地図がより使いやすくなればいいなと思ったのでありました．

2005-04-21

_ クラスファイルからの逆コンパイル

Javaはクラスファイルからソースコードに逆コンパイルできることは知っていたのですが，実際にやってみたこともなく，どうやるのかもしらなかったのですが，逆コンパイルする必要がでてきたので調べてみました．

JDKには標準でjavapというコマンドで逆コンパイルできます．

C:\> javap -c hogehoge

ただ，javapで逆コンパイルしたものはあんまり意味がわかりません．できれば元のソースがほぼそのまま再現されるのが一番便利です．これを実現してくれるツールにJadというものがあります．Jadで逆コンパイルをかけて出力されるファイルは，ほぼソースファイルを再現してくれます．ログ表示などに日本語などの２バイト文字を記述していると，Unicodeに変換されています．

コンパイルされたバイナリファイルはあるけどソースファイルが手元にないときは，このツールが役立ちそうです．
