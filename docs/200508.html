_ メモリ増設とkernelのアップデート

昨晩，サーバのメモリを256MBから512MBに増強しました．で，再起動ついでにkernelを2.6ベースにアップデート．woodyからそのままdist-upgradeしたsarge環境なのでブートローダーはliloを使っていたのですが，案の定というかなんというか，kernel panicで起動できず．

VFS: Unable to mount root fs on unknown-block(3,2)

といったメッセージで起動に失敗します．原因を調べるのも時間かかりそうだったので，ブートローダーをgrubに変更．

# apt-get install grub
# mkdir /boot/grub
# update-grub
# grub-install /dev/hda

これで無事起動できました．サーバの温度監視にlm-sensorsを利用しているのですが，kernelを2.6に変えたことでlm-sensorsを利用してグラフ表示しているHotSaNICの設定を変更する必要あり．HotSaNIC/modules/sensors/settingsを以下のように変更．

SENSOR=/sys/bus/i2c/devices/1-0290/temp2_input,temp2,CPU Temp,1,0.001,0,C

これでkernel2.6環境で温度の監視が可能になりました．

_ 洋楽歌詞検索プラグイン(2)

以前作成した洋楽歌詞検索プラグインのLetsSingIt検索プラグインはサイトの検索時にpost要求しか使えなくなっていました．他に洋楽の歌詞検索サイトでいいところはないかと探していたら，ありました．Lyrics Search Engineというサイトで，アーティスト名，曲名で検索ができます．

# Lyrics Search Engine

<search
       name="Lyrics Search Engine"
       description="All About Lyrics Search at Astraweb"
       sourceTextEncoding="1"
       method="GET"
       action="http://search.lyrics.astraweb.com/"

       queryEncoding="iso-8859-1"
       queryCharset="iso-8859-1"
>

<input name="word" user>
<interpret
       browserResultType="result"
       charset = "iso-8859-1"
>
</search>

毎度のごとく上のソースをそのままコピーしてもらってもいいですし，リンクからのインストールも可能です．

_ Firefoxでプラグインの追加の通知を表示させない

ZaurusにpdaXromをいれて，特にこれといった支障もなく快適に利用しているのですが，ひとつ覚書．Firefoxを利用しているとFlashを利用しているページなどでプラグインが未導入だと，プラグインを追加するように通知されます．MacromediaからはpdaXromで動作するFlashプラグインは提供されていないし，GPLFlashを使えばpdaXrom上のFirefoxでもFlashを再生できるようですが，今のところ必須ではありません．

この通知を消すにはどうしたものかと調べたところ，

Mozilla Firefoxのpluginsディレクトリからlibnullplugin.soを削除してください。

mozdev.org - プラグインのインストールを要求されないようにするにはどうしたらいいの？より引用

というわけで，libnullplugin.soを削除してみたわけですが，これだけでは通知は消えてくれませんでした．どうももうひとつ変更を加える必要があるみたいです．ロケーションバーに，about:configと入力して，plugin.default_plugin_disableの値をfalseにする必要があるみたいです．これらの設定をすることで，プラグイン導入の通知をさせないようにすることができました．

_ IEのアドレスバーの検索をGoogleに

ごくまれにIEで検索などをすることがあるのですが，Windows Updateのときぐらいしか使わないのでお気に入りとかもデフォルトのままです．ちょっと調べ物がしたいときに，アドレスバーで検索したい文字列を打ち込むと，IEではMSNサーチに飛ばしてくれます．でもやっぱり慣れ親しんだGoogleの検索のほうがいい．…ということで，IEのアドレスバーでの検索でGoogleを使う方法．

レジストリエディタを起動して，

HKEY_CURRENT_USER\Software\Microsoft\Internet Explorer\SearchUrl

の（既定）の値を

http://www.google.co.jp/keyword%s

とし，providerの値を"gogl"と設定します．これでIEのアドレスバーで検索をしてみると検索エンジンがGoogleに変わります．一度ログアウトをして，再度ログインすれば設定が反映されます．

_ tDiary on Lighttpd with FastCGI

なんとなくApache以外のWebサーバでtDiaryを動かしてみたい衝動にかられて，最近よくみるLighttpdを使ってみました．tDiaryを単純なCGIとして動かすだけであれば，最低限の設定をすることで難なく動かすことができました．この時点でApacheとLighttpdのベンチマークを比較してみると，

# Apache
Requests per second:    1.15 [#/sec] (mean)

# lighttpd
Requests per second:    1.17 [#/sec] (mean)

という結果で，まぁそう大差のない結果となりました．これではあまり面白みがないので，FastCGIを使ってどのくらい高速化できるかを試してみました．aptでlibfcgi-rubyをインストールして，LighttpdのFastCGIのモジュールを有効にします．設定は以下のようにしてみました．

fastcgi.server = (
        ".fcgi" => ( "localhost" =>
                (
                "max-procs" => 10,
                "socket" => "/tmp/tdiary.socket",
                "bin-path" => "/var/www/html/index.fcgi"
                )
        ),
)

呼び出すindex.fcgiは以下のようにしました．

#!/usr/bin/env ruby
$MY_DIR = File.dirname(__FILE__)
$LOAD_PATH.unshift $MY_DIR
Dir.chdir( File.dirname(__FILE__) )

require 'fcgi'

FCGI.each_cgi { |cgi|
        begin
                ENV.clear
                ENV.update(cgi.env_table)

                $CGI = cgi
                def CGI::new; $CGI; end

                load File.join($MY_DIR, 'index.rb')
        ensure
                class << CGI; remove_method :new; end
        end
}

上記コードは，tDiaryをFastCGI化とLighttpdAndFastCGIを参考にしました．LighttpdAndFastCGIでも書かれていますが，

よくわかんないけど、lighttpdでFastCGIを実行すると、カレントディレクトリが '/' になる。

という影響からなのか，tdiary.confが読み込めないエラーが発生しました．これを解決するために，「Dir.chdir( File.dirname(__FILE__) )」を記述しています．このサーバではtDiaryを複数設置構成でインストールしてあるので，index.rbはシンボリックリンクになっています．で，これでベンチマークをとってみると…

# FastCGI
Requests per second:    2.11 [#/sec] (mean)

数値上は早くなったみたい．体感ではいまいちよくわからないですが…．ベンチマークはとっていませんが，PHPをFastCGIで実行した結果は体感でも早くなったような気がします（気のせいかもしれませんが…）．もうちょっといろいろさわってみて，問題なさそうならApacheからLighttpdに移行してみようかと思います．

_ フライパンで秋刀魚

夏も終わりに近づいてきてそろそろ秋刀魚の季節．やたらと秋刀魚が食べたくなったのですが，アパート住まいで，コンロもないので焼くことができません．そういえば少し前にテレビでフライパンにクッキングシートを敷いて，その上で魚を焼くってのをやっていたのを思い出して試してみました．

お店でクッキングペーパーが売られているそのすぐ横に，フライパンで魚を焼くときに…と書かれた，そのものずばりの商品が．あとは秋刀魚1匹を100円で購入．おいしい秋刀魚の見分け方は，見た目の新鮮さ度合いはもちろんのこと，背中から見たときにまるまると太っているのが，脂ののったものだそうです．

さて，実際に調理するには，フライパンに収まる程度に切る必要がありました．とりあえず頭だけ切って，塩をふっておきます．火は中火から弱火くらいで焦げないように気をつけながら，両面をしっかり焼きます．15から20分もあれば中までしっかり火がとおったおいしい秋刀魚の塩焼きの完成です．

煙もでないので一人暮らしには十分な調理方法ですね．でも，やっぱりあの魚独特の生臭さが少しフライパンや蓋に残ってしまいました．この辺が何かいい回避方法はないものか…．とりあえずおいしく焼けることはわかったので，これからは家で魚を食べられそうです．
