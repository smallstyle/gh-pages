_ OpenSearch Autodiscovery に対応しました

Firefox を 1.5 系から 2.0 系に移行したことから，せっかくなのでこのサイトも OpenSearch Autodiscovery に対応させてみました．

OpenSearch に対応することにどのような意味があるのかは，朝顔日記 - 朝顔日記を OpenSearch Autodicovery 対応にに詳しく解説されているので省略．Firefox 2.0 などでは検索バーのところに変化があり，SmallStyle のサイト内検索プラグインをインストールできるようになります．

とりあえず，OpenSearch 方式の description ファイルを作成．(smallstyle_search.xml)

tDiary の plugin は，しばたさんがすでに作成されていたのでそのまま流用…と思ったら，9 行目の if 文でつかまってしまった．設定前にプラグインが有効になった場合，oepnsearch_xml が null なため，length 判定ができないようで…．とりあえず，

if opensearch_xml != nil then

に書き換えて対処．opensearch_ad_init が呼ばれて，文字列長で判定というのが本来のつくりだったのかな…．

追記：Autodiscovery に title 属性がないと，Firefox の検索エンジンの一覧で，ヘッダーの title 要素が利用されるので，ちょっとかっこ悪い感じがしたので，plugin に title 属性を追加した．

2006-06-08

_ Google Browser Sync を試してみる

Google が Firefox のブックマーク，履歴，cookie，パスワードを複数台のマシンで共有できる拡張 Google Browser Sync を公開していたようなので，さっそく試してみることにしてみました．

インストール自体は普通の拡張と同じように，インストールすることができます．Firefox を再起動すると，Google Browser Sync の初期設定画面が表示されます．ここでは，Google Acount の登録，PIN コードの登録，共有するデータの種類を選択できます．

設定が完了すると，Firefox が起動され，右上に Google Browser Sync のアイコンとユーザー名が表示されます．

手元に別のマシンがある場合は，そのマシン上の Firefox にも，Google Browser Sync をインストールしてみましょう．2 台目以降は，Google Acount を入力すると，最初にインストールしたときに指定した PIN コードの入力を求められます．ここで，PIN コードを入力すると，サーバにアクセスし，共有情報を引っ張ってきてくれるようです．

すでに，同一アカウントで Firefox が起動されていると，あとから起動した Firefox のアカウントが有効となり，先に起動していた Firefox の Google Browser Sync がサーバから切断されます．

この拡張の便利なところは，履歴やブックマークを共有できることと，最後に開いていた Window やタブを記憶していて，別のマシンで立ち上げたときに，再度開くことができることです．これは結構便利な機能かも．

会社などでいくつか開いておいたサイトを，家に帰ってからも続けてみたいときには，この機能を使えば拡張が自動的にデータを持っていてくれます．

パスワードや cookie などは暗号化されてはいるもののなんとなく気持ち悪いので無効化していますが，結構便利な拡張かも．起動時にデータの更新情報を取得しに行くので，その分ブラウザの起動が遅くなることがちょっとやだなぁ…

2006-06-15

_ パイプを使った正規表現は遅いらしい

qootas.org/blog - perl regex performance で，

"|"(パイプ)を使った正規表現はめちゃくちゃ遅いから使わないように、ということです。確かにベンチマークを取ると32倍速いです。

とのことだったので，Ruby ではどうだろうと試してみた．

#!/usr/bin/env ruby
require 'benchmark'

text = ';lskjdf;klvckxv;zijxcv;oa;vlkaefiuqewizlkvnzlxkcnv'
text << '.z,xmc v/z.x,cmv.z,xnvlafda isjdnfl aksjdfauerfaie'
text << 'jnlfakjdsn;akj;v akjdfvoaijdhfvoiaheriufahpsdiufhaeuhr'
text << ' iuahriufhairuhfapsidfalksjfhaiuphrofiankfjas;dofha[s9'
text << 'hfskjdf;ase;f,sedhfaiuwhefs,dnvflk dfis fapoisf fqjr'

n = 500000

Benchmark.bm(10) do |x|
   x.report("00_pipe:")    { n.times do ; /abc|def|ghi/ =~ text; end }
   x.report("01_nopipe:")  { n.times do ; /abc/ =~ text or /def/ =~ text or /ghi/ =~ text; end }
end
$ ruby regex.rb
                user     system      total        real
00_pipe:   15.703000   0.000000  15.703000 ( 15.735000)
01_nopipe:  3.297000   0.000000   3.297000 (  3.297000)

Ruby でも同様にパイプを使わない方が速いですね．4 倍以上といったところ．perl ほどの差は出なかったけど．極力使わないように気をつけよう．

_ コメントフォーム非表示プラグイン

以前，CSS を利用したコメントフォーム非表示プラグインを書いたのですが，フォームそのものが残っているのは，あまり意味もないかなぁということで，ML でもちょうど話題にあがっていたので，コメントフォームを生成しないプラグインを作成しました．

コメントフォームを非表示にしても，caption で「ツッコミを入れる」が残っているとかっこ悪いので，「ツッコミを読む」というふうに変わるようになっています．tDiary 内の変数で，hide_comment_form が定義されているのですが，プラグイン名が hide_comment_form.rb はやっぱり紛らわしいですかね？

ひとまずさっくり作ったものを公開してみます．hide_comment_form.rb を plugin ディレクトリへ．言語リソースのja，en，zh もそれぞれインストールしておいてください．言語リソースの入れ忘れには注意してください．

  • hide_comment_form.tar.gz

コメントフォームを非表示にするのは spam フィルタで指定された，n 日前の日記へのコメントは spam とみなすで指定された日記が対象になります．

今わかってる問題点は，「ツッコミを読む」リンクをクリックしたときに，コメントフォームが name 属性をもっているため，フォームを生成されていないと，ページ内のリンクが動作しない点．これは，また時間ができたら考えてみるつもり…

店についたら店の人が幹事にあいさつしたいとか，なんとかで，テーブルを担当するお姉さんから名刺をもらう．こういうサービスは顧客に安心感を感じさせて好感度アップ．この店は予約の電話時の応対も気持ちいいので，機会があればまた利用したいなと思えます．

食べ物のほうは今回はちょっと辛めなものが多くて，一部，香辛料が苦手な人にはちょっと悪い感じがしたけど，おおむね楽しんでもらえたようでなにより．

そろそろ飲み会のしきりも若い世代に引き継いでいかないとなぁ． 集金係を今年の新人にまかせみたけど，しっかりやってくれていたので，次回以降いろいろと任してみても問題なさそうだな．細かい部分での気配りが足りないのは，自分自身もできないから教えられないんだよなぁ…

2006-06-30

_ 温泉へ行こう(1)

ちょっと気分転換ということで，下呂温泉まで 1 泊旅行に行ってきました．今回の旅のコースは，郡上八幡，下呂温泉，飛騨高山をそれぞれ訪れることになりました．

天気は小雨が降ったりやんだりの天気．この時期，すかっと晴れることもない時期だし，晴れたら晴れたで暑そう．土砂降りなのも大変なので，それなりによかったかも．

まずは郡上八幡をめぜして国道 156 号線を北上．平日ということもあり，車の流れもスムーズ．10 時過ぎに出発して，昼過ぎには到着しました．昼食は，「大和屋」で，おろしそばを．めんは若干かためでなかなかおいしかった．

その後，日本の名水百選の第 1 号，宗祇水を見たり，小道をうろうろと 1 時間ほど散策．古い町並みや，雨にぬれた石畳は落ち着いた感じでのんびりとできました．観光客も他にほとんどいなかったし．

せっかく温泉にいくのだから早めに宿へ，ということで，16 時前に下呂到着を目指して郡上八幡を出発．とりあえずナビに旅館の場所を登録して，経路探索．途中ぐねぐねと曲がる山道はあるものの，道幅は十分あったので，らくらくと移動．途中の道の駅 明宝で，明宝ハムのフランクフルトを食べてみた．けっこうおいしかったのでお土産に買って帰ろうかとも考えたけど，冷凍だし，帰りにでも買えばいいかと思ったら，道の駅では明宝でしか買えないみたい（普通の明宝ハムは，売ってるんだけど）．

本日のお宿は，「水明館」．チェックインして部屋で，最初に抹茶と和菓子が出てきたのには驚き．チェックイン後，下呂市街を散歩して，夕食前にまずは温泉へ．水明館には，檜風呂，大展望風呂，野天風呂と 3 つの風呂がありました．広い上に，ちょうど自分がはいったときは人も少なくて，のんびりと温泉を楽しむことができた．お湯はちょっとぬめりけがある感じでした．夕食は部屋食で．目で楽しめ，味で楽しめるおいしい料理でした．

ちなみに下呂温泉では，河原にある露天風呂「噴泉池」があるのですが，下呂大橋のすぐ下に見えます．ちょうど散歩しているときに，集団でこの温泉にむかう人たちがいたのですが，宿に戻るときに「噴泉池」のほうに目をやると…（ちなみにまだ時間は 17 時過ぎなので明るかったのですが…）
