_ Safari で ThinkPad のトラックポイント・スティックを使ってスクロール

Safari for Windows beta が公開されて，少しさわっていたところでやっぱり気になるのが，ThinkPad についてるトラックポイント・スティックでスクロールができないこと．これが動かないと快適ブラウジングには程遠いということで，その対応策．Firefox のときでも，横スクロールができなくてその対応策を書いたわけですが，同じような手順で対応することができます．

C:\WINDOWS\system32\tp4table.dat のファイルをエディタで開いて，

; Pass 0 rules (These rules run first)

と記述されている部分があります．いくつかのアプリケーションの設定が見られると思います．そこに以下を追記します．

; Safari web browser
*,*,Safari.exe,*,*,*,WheelVkey,0,9

変更を保存したら，マシンを再起動してうまく動作するか確認してみてください．ちなみにタスクマネージャーで tp4serv.exe を終了して，新しいタスクの実行で，tp4serv.exe を起動してやっても変更が反映されるみたいです．

Safari にはいくつかのセキュリティホールがあることが取り上げられていたりするので，常用するわけにはいきませんが…(version 3.0.1 で修正されているみたいです）

_ Safari for Windows beta を試してみた

Windows 版 Safari の Beta 版がでたようなので，さっそく試してみた．見た目は iTunes 同様，Windows っぽくない感じ．日本語が表示されない…とか思ってたら，最速インターフェース研究会::Safari for Windows betaで日本語表示する方法で対応方法があった．とりあえず表示できればいいや，ということで，

手間をかけずに日本語表示を試したい人は

Documents and Settings\username\Application Data\Apple Computer\Safari\WebKitPreferences.plistを編集してフォント名を指定している箇所を全て「MS UI Gothic」に変更すると良さげ。

最速インターフェース研究会::Safari for Windows betaで日本語表示する方法より

上記を試してみた．うまく日本語は表示されたけど，やっぱり Beta 版といったところ．（メニューが崩れてしまったり，一部文字が表示されなかったり，途中で落ちたりなどなど）動きはけっこうさくさくと動いているので，今後がなかなか楽しみでもあります．
